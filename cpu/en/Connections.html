What are external connections?

The microprocessor has an array of metallic elements that enable the electrical connection between the integrated circuit that makes up the microprocessor and the circuitry on the motherboard. Depending on the complexity and power, a processor can have from 8 to more than 2000 metallic elements on the surface of its package. The processor is mounted with the help of a CPU socket soldered onto the motherboard. Generally we distinguish three types of connection:

  • PGA(Pin Grid Array):The connection is made by means of small metal wires distributed along the base of the processor, which are inserted into the motherboard through small holes. When the processor is inserted, a lever will anchor the pins so that they make good contact and do not come loose.
  • BGA(Ball Grid Array): The connection is made by means of balls soldered to the processor, which make contact with the socket.
  • LGA(Land Grid Array):The connection is made via smooth contact surfaces with small pins provided on the motherboard.

Processor busses

All processors have a main or system bus over which all data, instructions and addresses are sent and received from the chipset's integrated or other devices. As the connecting bridge between the processor and the rest of the system, it defines much of the system's performance, its speed is measured in bits per second.

There are 3 types of processor buses:

Data Buses

It carries the data that is transferred between units. It is usually bidirectional, i.e. the same wires are used to transmit information in or out of a unit, but always at different times. The difference between the internal data bus and the external data bus should be clarified: The internal data bus is used to transfer data between the elements of the mainframe (consisting of the CPU together with the main memory). On the other hand, the external data bus is considered an extension of the internal data bus as it connects the processor to the rest of the units.

Steering buses

Carries the address of the memory location or peripheral involved in the information traffic. It enables communication between the processor and the RAM memory cells. When the processor wants to read the contents of a memory cell, it sends the address of the cell it wants to read over the address bus and receives the contents of the cell over the data bus. The size of this bus defines the amount of RAM that the CPU can manage.

Control buses

This set of signals is used to synchronise activities and transactions with system peripherals. Some of these signals, such as R/W, are signals that the CPU sends to indicate what type of operation is expected at that time. Peripherals can also send control signals to the CPU, such as INT, RESET, BUS RQ. The most important signals on the control bus are the timer signals, which generate the time intervals during which operations are performed. This type of signals depends directly on the type of the microprocessor.
