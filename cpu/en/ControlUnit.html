What is the Control Unit?

The control unit (CU) is one of the three main functional blocks into which a central processing unit (CPU) is divided, is one of the three main functional blocks into which a central processing unit (CPU) is divided. The other two blocks are the processing unit and the input/output bus. It basically works by fetching instructions from the main memory, then "interpreting" (decoding) them, and executing them using the processing unit.

There are two types of control units:

  • Wired: generally used in simple machines.
  • Micro-programmed: more complex machines

In the first case, the main components are the sequential logic circuit, the state control circuit, the combinational logic circuit and the control signal recognition emission circuit. In the second case, the microprogramming of the CU is stored in a micro-memory, which is accessed sequentially to subsequently execute each of the micro-instructions.

Operations performed

Broadly speaking, it performs this cycle of functions:

  • Fetch: reads instructions from memory
  • Decode: Decode the instructions, and the memory addresses involved (if necessary).
  • Execute: Execute the instructions set by the program (sending the orders to the rest of the processor parts).

But moving on to a more detailed analysis, The functions performed by the control unit vary greatly by the internal architecture of the processor, as the control unit actually implements this architecture. On a regular processor that natively executes x86 instructions, the control unit performs the tasks of reading (fetching), decoding, execution handling and storing the results. In an x86 processor with a RISC core, the control unit has considerably more work to do: it handles the translation of the x86 instructions into RISC microinstructions, it handles the scheduling of the microinstructions among the various execution units, and it handles the output of these units to make sure they end up where they are supposed to go. In one of these processors the control unit is divided into other units due to the complexity of the work it must perform (such as a planning unit to handle scheduling and a withdrawal unit to handle the results coming from the pipeline). It stores the most frequently used data so that it is searched first in the computer and then in RAM.

Control Unit Components

Below you can read the components of the Control Unit:

  • Instruction register: it is in charge of storing the instruction being executed.
  • Program count: register containing the memory address of the next instruction to be executed.
  • Controller and decoder: it is in charge of interpreting the instruction for further processing. It is in charge of extracting the operation code from the instruction in progress.
  • Sequencer: generates micro commands necessary to execute the instruction.
  • Clock: provides a succession of electrical impulses at constant intervals.
  • Multiplex: Selects among several inputs and redirects the selection.

Types of CU

Mainly we have two types of control units, the wired and the microprogrammed:

  • The wired control units are used in very simple processors, since in them the table of registers and instructions cannot be changed since all the data paths are previously wired in a fixed way, in a wired control unit the instructions will always have the same performance and it will not be possible to add new ones or optimize its operation.
  • In the case of microprogrammed control units, a processor is used to perform all the functions of the control unit, which makes it possible to optimize the way in which the instructions are executed and even to add new instructions.

Due to the complexity of the microprogrammed control unit, they perform the decoding of the instructions much more slowly, which is why, depending on the type of instruction, one type of control unit is used or another when dealing with each of the instructions. Microprogrammed units usually consist of a microcontroller inside the processor. Their main advantage is that the manufacturer can optimize how each instruction is captured and decoded or even make changes if there is a security error in the way certain instructions are executed. This is key for when there are serious threats that can exploit the weak points of the CPU such as what happened with Spectre and Meltdown.
